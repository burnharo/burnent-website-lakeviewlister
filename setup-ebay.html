Setup / eBay Developer Setup

Setup guide

Connect your eBay developer credentials.

Lakeview Lister expects three main values for eBay OAuth in the app: Client ID, Client Secret, and an environment-specific RuName. After those are saved, the app opens the eBay consent flow for the seller account you want to manage.

Before you start

Use production first

For normal seller onboarding, create the production keyset and production RuName first. Only repeat the process for sandbox if your build actually exposes a sandbox environment.

Step 1: Join the eBay Developers Program and create a keyset

  1. Sign in to the eBay Developers Program.
  2. Open the Application Keys page.
  3. Create a Production keyset for the application you want to use with Lakeview Lister.
  4. Copy the Client ID and Client Secret from that keyset.

Create the eBay API keysets · Understand application keysets

Step 2: Create the RuName for that same environment

  1. On the keyset page, open the User Tokens area next to the Client ID.
  2. Create a RuName if one does not already exist for that keyset.
  3. Fill in the RuName form values needed for user-token OAuth.
  4. Copy the resulting RuName exactly as eBay shows it.
Use these callback URLs in the RuName form

Auth Accepted URL: https://flipforgeebaycallback.z19.web.core.windows.net/ebay-callback/

Auth Declined URL: https://flipforgeebaycallback.z19.web.core.windows.net/ebay-callback/declined/

Getting your redirect_uri value / RuName · eBay authorization guides

Step 3: Paste the values into Lakeview Lister

  1. Open Profile in Lakeview Lister.
  2. Go to the eBay OAuth 2.0 section.
  3. Paste the Client ID, Client Secret, and RuName.
  4. Save the profile if needed.
Environment matching matters

The RuName must match the same eBay environment as the keyset you created. Do not mix a Production keyset with a Sandbox RuName.

Step 4: Complete the OAuth consent flow

  1. Inside the app, click Open OAuth Consent.
  2. Sign in to the eBay seller account you want Lakeview Lister to manage.
  3. Approve the requested permissions.
  4. Return to the app and confirm that token status now shows valid access and refresh tokens.
  5. Use Refresh Token later if you need to renew access.

Optional sandbox note

If your build exposes a sandbox or developer-mode environment selector, repeat the same process with a separate sandbox keyset and sandbox RuName. For most normal seller setup, start with production first.
